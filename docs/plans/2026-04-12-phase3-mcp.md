# Steno Phase 3 — MCP Server Implementation Plan

**Goal:** Implement `steno serve` — a stdio MCP server exposing `steno_compress`, `steno_decompress`, and `steno_stats` tools. Compatible with Claude Code, Cursor, Windsurf, and any MCP-compatible client.

**Approach:** Use the official `rmcp` Rust MCP SDK (modelcontextprotocol/rust-sdk). Async runtime via `tokio`.

---

## Task 1: Add MCP dependencies

**Modify: `Cargo.toml`**

```toml
[dependencies]
# existing...
tokio = { version = "1", features = ["full"] }
rmcp = { version = "0.1", features = ["server", "transport-io"] }
serde_json = "1"
```

Verify: `cargo build`

---

## Task 2: MCP server module

**Create: `src/mcp/mod.rs`**
**Create: `src/mcp/server.rs`**

`server.rs` implements a `StenoServer` struct that implements the `rmcp` `ServerHandler` trait, registers 3 tools, and handles tool calls by invoking `build_codec()`.

### Tool schemas

**steno_compress:**
- Input: `{ "text": string }`
- Output: compressed text string + ratio

**steno_decompress:**
- Input: `{ "text": string }`
- Output: original text string

**steno_stats:**
- Input: `{ "text": string }`
- Output: `{ "original_bytes": int, "compressed_bytes": int, "saved_percent": float }`
